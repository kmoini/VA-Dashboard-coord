When the user asks to "add a checkpoint" (or "checkpoint"), perform ALL THREE steps, every time:

1. **Commit** — message must include the checkpoint number and a brief title, e.g. `checkpoint-002: <brief title of current state>`. If the working tree is clean, make an empty commit (`git commit --allow-empty`) so the checkpoint still gets its own numbered, titled history entry.
2. **Tag** — annotated git tag named `checkpoint-NNN` (zero-padded 3 digits, sequential). Find the next number with `git tag -l "checkpoint-*" | sort -V | tail -1`.
3. **Push** — push both the branch and the tag: `git push origin main` then `git push origin checkpoint-NNN`.

**Why:** The user wants restorable safe points so they can roll back if later modifications go wrong. A tag alone isn't enough for them — they explicitly want commit + tag + push together.
