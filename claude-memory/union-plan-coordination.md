- **Agreed sync model:** dashboard owns its PostgreSQL, **polls** mobile's new `/changes-since` endpoint per accountant, stores full row snapshots in `*_revisions` + `*_mirror` tables. Mirroring is **one-way mobile→dashboard**; clients do NOT see dashboard data through the app and do NOT log into the dashboard (so the dashboard's `type='client'` portal login is vestigial under this plan). Dashboard writes back only via `POST /accountant/send-message-to-user` and `/accountant/issue-invoice`. Invoice payment + detail pages stay dashboard-side (Stripe).
- **Linking keys:** mobile `users.id` (BIGINT) → `clients.mobile_user_id`; dashboard `accountant_accounts.id` (UUID) is source of truth for accountant login, stores mobile `accountants.id` as `mobile_accountant_id`.
- **ADR-0001 (open, awaiting Amin + Shahab):** M2M auth = asymmetric service JWT (dashboard signs with private key, mobile validates with public key, `accountantId` claim). No integration code until briefs lock + ADR-0001 approved.
